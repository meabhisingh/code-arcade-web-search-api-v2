API Documentation

Welcome to our API Service! Below you'll find information about the available endpoints, how to use them, and how to test if the service is working correctly.

Available Endpoints

POST /api/v1/search

Search for information using our powerful search engine.

Request Parameters

Parameter Type Required Description
query String Yes The search term or phrase to look for

Example Request

{"query": "machine learning algorithms" }

Example Response

{
    "success": boolean,
    "meta": {
        "timeTaken": string,
        "length": number,
        "size": string
    },
    "results": [{
        "source": {
            "url": string,
            "title": string,
            "website": string,
            "icon":"string"
        },
        "content": string
    }]
}

GET /api/v1/test

Simple endpoint to verify the API service is running properly.

Example Request

Simply send a GET request to /api/v1/test

Example Response

Test route works!

Using the API

With cURL

# Search API Example
curl -X POST http://localhost:8080/api/v1/search
-H "Content-Type: application/json"
-d '{"query": "your search term"}'

Test API Example
curl -X GET http://localhost:8080/api/v1/test

With JavaScript

// Search API Example
fetch('http://localhost:8080/api/v1/search', { method: 'POST', headers: { 'Content-Type': 'application/json', }, body: JSON.stringify({ query: 'your search term' }) }) .then(response => response.json()) .then(data => console.log(data));

// Test API Example fetch('http://localhost:8080/api/v1/test') .then(response => response.json()) .then(data => console.log(data));

Error Handling

The API returns appropriate HTTP status codes:

  • 200 - Success
  • 400 - Bad Request (missing or invalid parameters)
  • 404 - Not Found
  • 500 - Server Error
